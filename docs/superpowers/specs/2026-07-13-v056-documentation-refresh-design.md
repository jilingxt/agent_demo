# v0.56 说明文档刷新设计

## 目标

四份文档使用同一份当前事实，但面向不同读者展开；保留 v0.51 历史基线，删除或改写已被 v0.56 当前实现推翻的现状表述。

## 内容分工

- `README.md`：当前能力、最短运行路径、输出物、边界和文档导航。
- `项目介绍.md`：建设目标、法理三层抽象、案件中立能力、典型数据流、价值和边界。
- `技术手册.md`：模块职责、数据契约、调用顺序、传参、关系组件、主观证据、权威锚定、贝叶斯安全弃权、RAG 和验证。
- `用户手册.md`：准备材料、运行命令、EvidenceBook 阅读、结果状态、弃权处理、法律候选、常见问题和人工复核。

## 清理规则

明确标记为 v0.51 历史基线的内容保留；没有历史标记、却将五类回放、两部法律、人工确认案件类型或旧测试数量描述为当前状态的内容必须删除或改写。当前状态只在一个连续的 v0.56 主章节中表述，避免依靠后续“补充校正”反转前文。

## 事实来源

文档中的版本、模型 ID、弃权原因、RAG 规模、命令和字段必须从 `pyproject.toml`、`config/bayesian_models/registry.json`、`case_agent_demo` 实现、`legal_knowledge/metadata/corpus_manifest.json` 和实际验证输出取得。
