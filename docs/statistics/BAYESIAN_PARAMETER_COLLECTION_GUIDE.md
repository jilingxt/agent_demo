# 贝叶斯参数采集与人工更新指南

本指南配套使用 [`bayesian_parameter_collection_template.xlsx`](./bayesian_parameter_collection_template.xlsx)。对象是没有接触过统计学的业务人员、复核人员和模型管理员。

工作簿只保存去标识化统计数据。不得填写姓名、证件号、电话、住址、材料原文、图片、音视频或可识别个人的文件路径。

## 一、先回答最常见的问题

### 1. 报警人、嫌疑人等角色的笔录真实性比例，是不是后验概率

**可以用真实数据计算该统计分组长期表现的 Beta 后验，但它不是某个新案件事实的单案后验概率。**

例如统计口径为：

```text
数据批次：2026年6月
来源类别：statement
来源角色：reporting_person
谓词：conduct
来源立场：肯定
独立核验结果：真60、假20、未知2
```

采用最简单的 `Beta(1, 1)` 先验：

```text
后验 Alpha = 1 + 60 = 61
后验 Beta  = 1 + 20 = 21
后验均值   = 61 / (61 + 21) = 0.7439
```

`0.7439`表示：在明确限定的时间、样本、角色、谓词和立场分组中，对“长期肯定主张核验符合率”的估计。它**不表示**：

```text
新案件中报警人说的某个事实有74.39%概率为真。
```

新案件还要结合当前案件的独立证据、来源依赖、反向材料、权威材料和冻结的贝叶斯网络参数进行推理。

### 2. 能否直接按角色设置固定可信度

不能。`报警人`、`嫌疑人`、`被害人`、`证人`只是分层字段，不是信誉分。统计必须至少同时限定：

```text
数据批次 × 模型版本 × 谓词 × 来源类别 × 来源角色 × 来源立场
```

同一角色对不同事实的表现可能完全不同。角色统计只能形成候选的观测参数或校准依据，不能把某类人的全部陈述永久写成一个固定权重。

### 3. 人工能修改哪些参数

当前六个案件族模型共有 **60 个直接贝叶斯数值参数**：

| 参数 | 数量 | 能否由真实数据更新 | 正确方法 |
| --- | ---: | --- | --- |
| 根节点 `prior` | 27 | 可以 | 用独立核验的真/假计数做 Beta 后验，先形成候选值。 |
| 派生节点 `intercept` | 6 | 可以，但不能手算 | 与同一节点的全部 `weights` 一起做离线正则化逻辑回归。 |
| 派生节点 `weights` | 27 | 可以，但不能用单个正确率替换 | 收集逐案父节点值和子节点独立真值，整体拟合后验证。 |

项目中还有证据质量权重、来源类型默认质量、主观证据基础不确定量、权威锚定 `mean/strength` 等参数。这些可以在有足够独立标注和验证数据后**整体校准**，但不应按月根据一个比例直接覆盖。

以下内容禁止通过历史频率自动修改：

- 网络因果方向、案件族注册表和必需输入；
- 法定年龄、金额、次数、期限、证明标准和处罚边界；
- 罪名、责任、管辖、抗辩、例外和最终法律适用；
- 权威文书的专业适用范围和法律效力；
- “达到某比例就认定违法犯罪”等自动决策规则。

## 三、第一步：建立可统计的原子事实

一行只能记录一个可独立核验的事实。例如一份笔录同时出现“某人在场”“发生接触”“取得财物”，应拆成三行，而不是给整份笔录一个真假标签。

在`原子事实记录`中至少填写：

1. `样本ID`：不可逆内部代号，不含姓名。
2. `事件ID`：同一案件事件使用同一代号。
3. `数据批次`：例如`2026-06-unit-A`，同时另行保存口径说明。
4. `模型ID/版本`：必须与运行时模型一致。
5. `节点/谓词`：例如 `conduct`、`result_exists`，不得只写自然语言概述。
6. `来源材料ID`：去标识化代号。
7. `来源组ID`：同一底层材料及其派生件使用同一组。
8. `来源类别/角色`：从下拉框选择。
9. `来源主张标签`：来源肯定填`真`，否定填`假`，没有明确表态填`未知`。
10. `独立核验标签`：由独立核验人员填`真`、`假`或`未知`。
11. `是否纳入统计`：只有核验完成、口径一致且不重复的记录填`是`。

`未知`表示材料不足，不能改写为`假`。公式会自动形成：

- **TP**：来源主张真，独立核验真；
- **FP**：来源主张真，独立核验假；
- **FN**：来源主张假，独立核验真；
- **TN**：来源主张假，独立核验假；
- **未知**：任一关键标签未知；
- **排除**：未纳入统计。

## 四、第二步：更新根节点 prior

根节点 `prior` 表示在明确样本范围内、尚未输入本案证据前，该原子事实成立的基础水平。它不是违法犯罪概率。

### 收集什么

对每个根节点收集：

```text
模型ID、模型版本、节点、数据批次、独立核验真数、假数、未知数
```

未知不进入分母。同一事件、同一原始来源的重复材料不能重复计数。

### 怎么算

使用 Beta 后验：

```text
后验 Alpha = 先验 Alpha + 真数
后验 Beta  = 先验 Beta  + 假数
候选 prior = 后验 Alpha / (后验 Alpha + 后验 Beta)
```

工作簿默认使用保守的 `Beta(1, 1)` 示例先验，并计算 95% 区间。发布时不仅看均值，还要看：

- 有效样本是否足够；
- 真和假两类是否都有样本；
- 95%区间是否过宽；
- 验证批次是否优于旧参数；
- 是否存在案件来源、时间或单位分布变化。

模板中的`样本太少`、`试点`和`可进入独立验证`只是预警，不是自动发布命令。

## 五、第三步：统计不同来源的主张表现

`来源表现统计`用于回答更精确的问题，而不是给角色打总分。

| 指标 | 分子 | 分母 | 白话解释 |
| --- | --- | --- | --- |
| 肯定主张正确率 | TP | TP + FP | 来源说“成立”时，后来核验为真的比例。 |
| 否定主张正确率 | TN | TN + FN | 来源说“不成立”时，后来核验为假的比例。 |
| 灵敏度 | TP | TP + FN | 事实确实成立时，来源明确肯定的比例。 |
| 特异度 | TN | TN + FP | 事实确实不成立时，来源明确否定的比例。 |

每个指标都使用 Beta 后验和区间。必须同时保留未知数和有效分母。小样本角色不得单独发布，可合并到更高层来源类别，或继续保留专家先验。

这些统计可以用于：

- 检查当前证据质量默认值是否偏高或偏低；
- 为未来观测模型提供候选参数；
- 监测抽取器或来源流程在不同版本下是否漂移。

这些统计不能用于：

- 自动判定某个角色说真话或说假话；
- 直接替换某个贝叶斯 `weight`；
- 跳过本案其他证据和人工核验；
- 形成对具体人员的长期信誉档案。

## 六、第四步：拟合派生节点 intercept 和 weights

当前派生节点使用逻辑型 CPD：

```text
P(子节点成立) = sigmoid(intercept + weight1×父1 + ... + weightN×父N)
```

因此，`intercept`和`weights`不能从“正确率为74%”直接手工换算。必须在`CPD样本记录`中逐行收集：

```text
样本ID、事件ID、模型ID、模型版本、子节点、
父节点1名称和值、父节点2名称和值……、
子节点独立核验真值、数据批次、数据用途、复核状态、是否纳入拟合
```

父节点值可填 0 到 1 的软证据值；无法核验填`未知`。子节点标签必须来自独立核验。数据用途必须预先分成：

- `训练`：拟合参数；
- `验证`：选择参数、检查过拟合；
- `测试`：最后一次独立验收，不参与调参。

拟合时使用带正则化的二项逻辑回归，并一次性拟合同一子节点的截距和全部父节点权重。至少报告：

```text
训练/验证/测试有效样本量
真/假类别数量
缺失与排除数量
拟合系数和标准误或区间
验证集对数损失、Brier分数或校准结果
与旧版本的回放对比
```

`CPD组合统计`可以把离散父节点组合的真/假数做 Beta-CPT，帮助人工理解和发现异常组合，但它不能直接生成逻辑回归权重。

## 七、来源依赖与去重

以下材料通常不是独立来源：

- 同一视频的多张截图；
- 复制或转发的同一图片；
- 报告引用同一份笔录或同一段监控；
- 多人互相转述同一来源；
- 同一制作流程生成的多个派生文件。

这些记录应使用相同`来源组ID`，并在`来源依赖`中登记原始来源、派生来源和依赖类型。统计时先在同一来源组内去重或取最强有效观察，再跨独立来源组累积。十张同源截图不能等于十个独立证人。

## 八、抽取质量、证据质量和权威锚定

### 抽取质量

`抽取质量统计`评价模型有没有正确读出材料：

```text
precision = TP / (TP + FP)
recall    = TP / (TP + FN)
F1        = 2 × precision × recall / (precision + recall)
```

它不表示材料内容真实。图片文字识别很准确，不等于图片来源真实或与案件相关。

### 证据质量权重

项目中的证据质量维度权重和来源类型默认质量应使用带人工标注的 Claim 评估结果整体校准。不能把某个月的角色正确率直接复制到某个质量权重。没有稳定标注、验证集和回放结果时，应保留当前专家配置。

### 权威锚定

法医、鉴定、价格认定等专业结论应先审查资格、真实性、程序、对象对应、方法、标准和专业范围。`权威锚定复核`中的历史维持率只用于监测：

- 是否频繁出现相反重新鉴定；
- 是否存在程序无效或对象不对应；
- 当前 `authority mean/strength` 是否需要专家重新评估。

不得看到维持率为 97% 就自动把锚定 `mean` 改成 0.97。权威锚定还承载法律和专业规则，必须由专业、法律和模型人员共同审批。

## 九、人工修改参数的安全流程

1. **冻结数据口径**：记录起止日期、纳入排除条件、模型版本和数据批次。
2. **完成独立核验**：未知、分歧、同源依赖和重复记录均已处理。
3. **生成统计结果**：根节点使用 Beta 后验；逻辑 CPD 使用离线整体拟合。
4. **登记候选值**：在`候选参数`填写当前值、候选值、样本量、方法、区间和验证结果。
5. **不得覆盖旧模型**：复制当前 JSON 为新版本，例如从 `*_v1.json` 创建经批准的新版本文件。
6. **修改准确路径**：工作簿已列出 JSON 路径，例如 `nodes[id=conduct].prior` 或 `nodes[id=causation].weights.conduct`。
7. **保持安全约束**：不得删除 `required_inputs`、案件族注册、方向限制、弃权和法律边界。
8. **运行完整回放**：比较旧版与候选版，重点检查伤害以外案件、未知案件、替代原因和反向证据。
9. **独立测试**：测试数据不得参与拟合和阈值选择。
10. **记录发布信息**：保存模型版本、参数哈希、校准状态、数据截止日、审批、隐私检查和回滚包。
11. **通过后再切换**：只允许`已批准`且隐私检查`通过`的版本进入运行配置。
12. **可随时回滚**：旧版本和历史运行所用参数哈希必须保留。

运行案件时只读取冻结参数，不在单案流程中在线训练或自动改写模型文件。

## 十、样本量和质量底线

模板使用以下提示帮助非统计人员发现明显问题：

- 有效样本少于 30：只描述，不发布候选参数；
- 任一真/假类别少于 10：类别不足，不做稳定拟合；
- 达到 100：也只是可以进入独立验证，不代表必然可发布；
- 逻辑回归需要的样本量随父节点数量、类别不平衡和缺失情况增加；
- 95%区间很宽时，即使均值看起来合理，也应继续收集数据。

没有适用于所有节点的统一最小样本量。最终应由统计复核人员结合区间、验证集表现、数据漂移和业务风险判断。

## 十一、隐私与职责分离

1. 数据整理人员生成不可逆样本 ID，不复制材料原文。
2. 模型运行人员填写模型输出或来源主张。
3. 核验人员按批准规则独立填写真值，不查看模型置信度。
4. 分歧由另一授权角色裁决，保留初始标签和裁决状态。
5. 统计人员汇总并拟合，不负责同批数据的最终审批。
6. 模型、法律和隐私复核人员共同检查候选参数。
7. 发布人员只发布有完整记录、哈希和回滚包的版本。

小样本分组应合并或抑制，避免通过稀有组合重新识别人员。发现误录个人数据时应立即停止流转，隔离文件并重新生成去标识化统计。

