# 贝叶斯参数统计采集指南

本模板用于离线采集、复核和批准案件中立的贝叶斯模型参数。工作簿只保存去标识化统计和治理元数据，不得填写姓名、证件号、联系方式、住址、材料原文或其他可识别个人的信息。

## 工作表

| 工作表 | 用途 |
| --- | --- |
| `填写说明` | 数据边界、标签规则、发布边界和法律边界速查。 |
| `参数需求清单` | 说明网络需要的参数、真实数据需求、估计方法，以及禁止训练的法律参数。 |
| `原子事实核验` | 逐项记录模型标签与独立核验标签，并生成 TP/FP/FN/TN。 |
| `来源观测统计` | 按来源类别和谓词汇总真阳性率或假阳性率的 Beta 后验。 |
| `抽取准确率` | 汇总字段或谓词抽取的精确率、召回率和 F1。 |
| `来源依赖` | 标记共同来源、共同制作过程及其他非独立性。 |
| `案件族CPD` | 按父节点状态汇总子节点条件概率的 Beta 后验。 |
| `权威锚定复核` | 复核权威材料的版本、有效性和适用范围；不估计其“可信概率”。 |
| `模型发布记录` | 记录版本、参数哈希、校准状态、隐私检查和离线审批。 |
| `枚举值` | 为下拉列表提供受控值，不应随意改写。 |

## 标签和混淆矩阵

`模型输出标签`和`独立核验标签`使用`真`、`假`、`未知`。空白表示尚未核验，`未知`表示现有材料不足以判断。不得把空白或`未知`改写为`假`，也不得让它们进入成功数、失败数或比率分母。

以模型输出为预测、独立核验为参考：

- **TP（真阳性）**：模型为真，独立核验为真。
- **FP（假阳性）**：模型为真，独立核验为假。
- **FN（假阴性）**：模型为假，独立核验为真。
- **TN（真阴性）**：模型为假，独立核验为假。

`原子事实核验!G2`是示例公式。仅当两个标签都已确定为真或假时才返回 TP/FP/FN/TN；任一标签为`未知`时返回`未知`，任一标签为空时保持空白。后续数据行可复制该公式，但发布前应由统计脚本重新汇总并核对分母。

## Beta 后验

对概率参数使用 Beta-Binomial 更新：

```text
posterior_alpha = prior_alpha + success_count
posterior_beta  = prior_beta  + failure_count
posterior_mean  = posterior_alpha / (posterior_alpha + posterior_beta)
```

在`来源观测统计`中：真阳性率以 TP 为成功数、FN 为失败数；假阳性率以 FP 为成功数、TN 为失败数。在`案件族CPD`中，给定父节点状态后，以子节点真计数为成功数、子节点假计数为失败数。`未知数`单独报告，不参与后验更新。先验 Alpha 和 Beta 必须大于零，并记录其专家批准依据；工作簿公式是核对示例，不是在线训练程序。

当前模型中的 `prior` 可由根事实节点的真/假计数形成 Beta 后验；`logistic` 节点的 `intercept/weights` 需要使用`案件族CPD`中按父节点组合汇总的二项计数做离线加权逻辑回归，或在样本不足时继续保留专家先验。运行案件时只读取已经批准的冻结参数，不在单案流程中在线训练。

报警人、嫌疑人、被害人、证人等角色统计应按“角色×谓词×立场”形成观测灵敏度和假阳性率，并采用分层模型或部分汇聚处理小样本。角色本身不能直接转换成固定可信度，也不能把一个月内尚未独立核验的办案结论作为真实标签。

## 独立核验

1. 数据整理人员生成不可逆`样本ID`、`原子事实ID`和来源批次，不复制材料原文。
2. 模型运行人员填写模型输出标签，不向核验人员提供模型解释或置信度。
3. 核验人员依据批准的标注规则独立填写标签；不得仅因陈述人角色、职业或身份赋予固定可信度。
4. 对不一致项由另一名授权角色裁决，保留初始标签、分歧状态和角色代号。
5. 统计人员汇总 TP/FP/FN/TN、未知数和来源依赖；同源复制材料不能按独立观察重复计数。
6. 复核人员检查公式、样本边界、数据截止日期、参数哈希和隐私结果。

角色代号只表示职责分离，不得编码真实姓名。核验人员不能既生成模型标签又担任同一批数据的最终独立复核人。

## 来源、依赖和溯源

每条统计至少保留模型 ID/版本、数据批次、谓词或节点、来源类别、核验状态和截止日期。`来源依赖`用于记录共同底稿、转述链、复制件、共同制作过程、时间耦合或内容耦合。存在依赖时，应合并观察、降权或交由批准的依赖模型处理，不能把多个副本当作多名独立来源。

原始材料与统计工作簿应分库存放。工作簿仅记录受控类别、不可逆 ID 和必要的位置引用；位置引用不得包含用户名、网络共享凭据或可识别个人的目录名。

## 权威与法律边界

`权威锚定复核`只确认文书版本、发布日期、来源位置、现行有效性和适用范围。以下值是确定性的法律规则或经批准的专家先验，**不可从案件真值率或角色历史表现中学习**：

- 法定年龄、数额、次数、期限和法律门槛；
- 法律身份、职权、职责、授权范围及管辖；
- 抗辩、免责、例外、法定推定和证明标准；
- 行政与刑事边界、罪名、责任、处罚和最终法律适用；
- 仅基于陈述人角色设定的固定可信度。

统计模型只推断事实性谓词。不得输出有罪概率、处罚概率或最终法律适用概率。

## 隐私控制

- 采集前去标识化，仅保留完成统计所需的最小字段。
- 不录入自由文本证据、图片、音视频、精确地址、设备标识或人员关系图。
- 小样本分组应合并或抑制，避免通过稀有组合重新识别个人。
- 工作簿在受控离线环境流转，使用访问控制、完整性哈希和留存期限。
- 发布前执行人工隐私检查；结果为`未完成`或`不通过`时不得批准。
- 发现误录个人数据时停止流转，按内部事件流程隔离并重建去标识化统计。

## 离线版本审批

1. 锁定数据截止日期和数据批次，完成未知项、依赖项和分歧项复核。
2. 导出参数文件，记录模型 ID、模型版本、校准状态和 SHA-256 参数哈希。
3. 独立复核参数范围、Beta 分母、CPD 父状态覆盖及工作簿公式。
4. 完成隐私检查，并确认法律规则未被写入可学习参数。
5. 在隔离环境执行回放验证；未使用真实数据验证的版本保持`expert_prior_unvalidated`或`pilot_uncalibrated`。
6. 由授权角色离线审批。只有`已批准`且隐私检查`通过`的版本可进入发布包。
7. 保存发布包相对路径、替代版本和审批日期；旧版本保持可追溯，不覆盖历史记录。

生成或刷新模板：

```powershell
python scripts/generate_bayesian_statistics_workbook.py
```

生成后运行`python -m pytest tests/test_statistics_workbook.py -q`验证工作表、公式、筛选、冻结窗格和数据验证契约。
