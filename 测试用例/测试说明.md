# 多案件回放测试说明

## 一、问题结论

### 1. 为什么三份殴打材料仍被判断为证据不足

原流程没有把“陈述人”和“行为人”稳定区分，也把“被害人”和“伤情结果”拼在同一个 `object` 字段中。三份材料因此被拆成：

- `贺显作 -> 对方 -> violence`
- `李文杰 -> 贺显作 肿包流血 -> violence`
- `李文杰 -> 贺显作 -> violence`

每个 Claim 只有一个来源，主观证据模型自然给出 `insufficient`。法医报告虽然可以成为 `authority_anchored`，但其事件和实体未与行为 Claim 对齐，贝叶斯模型只能得到缺少 `result_exists` 等输入的低派生值。

本次修复后，Text/Report 的结构化输出明确包含：

```text
declarant / actor / target_person / object / predicate
event_id / stance / source_group / origin_evidence
```

原三份 DOCX 现在形成一个 `李文杰 -> 贺显作 -> violence` Claim，三个独立来源进入同一主观证据评估。

### 2. 为什么会匹配寻衅滋事、绑架和威胁等条款

原 RAG 使用一段包含事实、Claim、状态和英文原因的长查询，再以字符二元词、向量相似度和宽松 OR 门槛取满 `top_k`。例如“随意殴打他人”会被当作一般伤害高分命中，“故意伤害被绑架人”也会使绑架条款获得伤害领域分。

本次改为：

```text
支持的事实 Claim
  -> 行为族识别
  -> 在 law_DB 两部完整法律中筛选主行为族条款
  -> FTS5 + BGE 向量召回
  -> 关键词、向量、领域、精确短语综合重排
  -> top_k 仅作上限，不强行用异类条款填满
```

自定义规范文件不受两部法律的行为族硬筛选，仍可通过 hybrid RAG 召回。

上述贝叶斯值使用版本化专家先验，表示事实要素之间的支持强度，不是违法犯罪概率，也不替代刑事/行政边界和最终法律判断。

## 三、权威材料边界

法医鉴定只有在传入明确的人工有效性核验后，才对 `injury_grade` 形成权威锚定。核验包括主体资质、真实性、程序、被鉴定人对应、方法、标准、证明范围和人工确认。

轻伤二级权威成立不自动证明：

- 谁实施了行为；
- 该损伤必然由本案行为造成；
- 行为人具有何种主观故意；
- 已构成犯罪或应当处罚。
